# 匿名树洞项目：严格按课件执行指南与 TODO

## 1. 课件要求提炼

根据课件《Web开发技术基础课程设计》当前与本项目直接相关的要求，可以提炼出以下结论：

- 阶段一题目就是 `匿名树洞 / 校园表白墙 (Anonymous Wall)`。
- 阶段一主题是 `AI 驱动的快速开发`，时间安排为第 `1-3` 周。
- 课件给出的阶段一安排是：
  - 第 `1` 周讲解
  - 第 `2` 周答疑
  - 第 `3` 周检查
- 阶段二主题是 `生产级部署与上线`，时间安排为第 `4-6` 周。
- 课件在部署部分不是只讲概念，而是给出了明确的学习用落地路线：
  - `免费域名 + 非内地服务器`
  - 前端部署到 `OSS` 静态站点
  - 后端部署到 `Serverless / 函数计算`
  - 配置 `SSL`
  - 数据库不能继续依赖“跟代码绑在一起的本地 SQLite 文件”
  - 最终可以继续扩展到 `CI/CD`
- 课件同时强调：
  - 学习用方案可以不走正式备案
  - 如果是正式上线系统，仍应按规定备案

因此，严格按课件执行时，这个项目现在不应继续把重心放在“新增业务功能”，而应转向：

- 把阶段一现有成果整理成可检查、可演示的项目
- 再把它推进到课件阶段二要求的 `OSS + Serverless + SSL + 持久化数据库`

## 3. 严格按课件应如何完成

## 3.1 总体路线

严格按课件时，建议你不要再把路线选成传统单机 Docker + Nginx 作为最终答案，而要把它作为本地开发或过渡方案。最终交付路线应写成：

`匿名树洞功能完成 -> 前端打包 -> OSS 静态部署 -> 后端函数计算部署 -> SSL -> 持久化数据库 -> 演示验收`

也就是说，课件允许你了解多种部署模式，但它后续实际演示的是：

- 前端：`OSS`
- 后端：`Serverless / 函数计算`
- 域名：`免费域名`
- 服务器区域：`非内地`
- HTTPS：`SSL`

## 3.2 你现在应该先做什么

第一步不是买域名，也不是继续补新功能，而是先冻结范围。

你当前应该先确认以下内容已经能稳定演示：

- 注册 / 登录
- 匿名身份生成或刷新
- 首页帖子列表
- 发帖
- 评论与回复
- 点赞 / 收藏
- 举报
- 私信或通知
- 管理员审核

如果这条主流程不稳定，就先修稳定性，不继续扩功能。

## 3.3 前端应该怎么改

因为课件要求前端进入 `OSS` 静态托管，所以前端不能继续只依赖开发期的 Vite 代理。

当前仓库里：

- `frontend/src/api/index.ts` 仍写死为 `baseURL: '/api'`
- `frontend/vite.config.ts` 只配置了本地开发代理

严格按课件时，前端需要改成：

- 开发环境继续走本地代理
- 生产环境通过环境变量读取真实后端地址

推荐目标：

- 新增 `VITE_API_BASE_URL`
- 本地开发未配置时默认 `/api`
- 生产打包时显式注入函数计算的公网地址

这样你在课件要求的“打包前端项目，设置后端 BASE_URL”这一步就有明确落点。

## 3.4 前端上线应该怎么做

按课件路线，前端上线步骤应是：

1. 修改前端生产环境 API 地址
2. 执行 `npm run build`
3. 得到 `dist/`
4. 创建 OSS 静态站点
5. 上传 `dist/` 内容
6. 验证首页可访问
7. 验证前端能成功请求后端接口

这里要注意：

- 不是只看页面能打开
- 必须验证登录、列表、发帖等真实接口调用成功

## 3.5 后端上线应该怎么做

按课件路线，后端上线目标不是传统 VPS 常驻进程，而是 `函数计算 / Serverless`。

因此你需要完成：

- 明确 Django 在函数计算中的入口方式
- 准备部署包
- 准备依赖层
- 修改监听地址与入口配置
- 关闭不必要的触发器认证
- 查看日志，处理缺依赖、路由、入口错误

从课件展示的典型问题看，你需要重点防以下坑：

- 没装依赖包
- 依赖没有做成层
- 监听地址错误
- 网关签名认证未关闭
- 默认路由未处理

## 3.6 数据库必须怎么处理

这一步是严格按课件时很容易丢分的地方。

课件最后明确提醒：

- 如果数据库文件和代码在一起
- 那么实例重启时会重置
- 这不适合作为最终上线方案

因此你这个项目上线时不能把 `backend/db.sqlite3` 当正式部署数据库。

严格做法是：

- 切换为独立 PostgreSQL

如果临时演示环境无法拿到正式数据库，也至少要在文档中明确说明：

- SQLite 仅用于本地开发
- 上线环境改为网络数据库

## 3.7 HTTPS / 域名应怎么处理

按课件要求，学习用方案可以使用：

- 免费域名
- 非内地服务器
- SSL

所以你的最终交付不必强制购买正式商业域名，但必须把方案写清楚：

- 为什么不需要备案
- 为什么要选非内地节点
- 域名和 DNS 用什么服务
- SSL 怎么配置

如果最后你没有真的买域名，也建议至少做到以下两件事中的一种：

- 使用平台默认公网地址 + SSL
- 使用免费域名绑定前端和后端服务

## 3.8 CI/CD 要做到什么程度

课件第 70 页提到“使用应用中心，支持 CI/CD”。严格执行时，最理想是补一个最小自动部署链路。

如果时间有限，最低可接受版本建议是：

- 代码推送后自动构建前端
- 自动生成前端 `dist`
- 自动触发部署脚本或至少产出构建包

如果做不到完整自动部署，也要在文档中明确写出：

- 当前完成了哪一步自动化
- 哪一步还需要人工操作

## 4. 交付标准

如果要说“这个项目严格按课件完成了”，至少应该满足下面这些结果：

- 匿名树洞核心业务可完整演示
- 有固定演示账号和演示路径
- 前端已经能打包并部署到静态托管
- 后端已经部署到函数计算或同等 Serverless 方案
- 前后端公网联通
- 访问地址为 HTTPS
- 数据库不是临时易失的本地 sqlite 文件
- 有一份清晰的部署与验收文档
