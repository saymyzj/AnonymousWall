# AnonymousWall — 匿名树洞/校园表白墙 功能设计文档

## 一、项目概述

### 1.1 项目定位

面向校园用户的匿名社区 Web 平台，用户可匿名发布表白、吐槽、求助等内容，支持评论互动、匿名私信等社交功能。项目以网站形式呈现，兼容移动端浏览器。

### 1.2 核心特点

- **前台匿名，后台实名**：用户真实身份仅管理员在必要时可查看
- **三级内容审核**：敏感词过滤 + AI 审核 + 人工审核，保障社区环境
- **个性化推荐**：基于用户行为的推荐排序，提升内容消费体验

### 1.3 页面结构总览

```
前台页面：
├── 首页（卡片流，标签筛选 + 搜索 + 排序）
├── 帖子详情页（内容 + 评论列表 + 投票）
├── 发帖页（表单：内容 + 标签 + 背景色 + 图片 + 投票）
├── 个人中心（我的发布 / 评论 / 收藏）
├── 消息中心（通知 + 私信）
├── 登录 / 注册页

管理后台：
├── 仪表盘（数据统计）
├── 内容管理（帖子 / 评论）
├── 举报管理
├── 审核队列
├── 用户管理
├── 敏感词管理
├── 置顶与公告管理
```

## 二、用户体系

### 2.1 注册与登录

| 功能 | 说明 |
|------|------|
| 注册方式 | 邮箱注册，需验证邮箱有效性 |
| 登录方式 | 邮箱 + 密码登录 |
| 密码安全 | 密码强度要求（8 位以上，含字母和数字），后端 bcrypt 加密存储 |
| 登录状态 | JWT Token 维持登录态，有效期 7 天，支持续签 |

### 2.2 匿名身份

| 功能 | 说明 |
|------|------|
| 随机昵称 | 注册后系统自动生成随机昵称（如"匿名熊猫#3927"），前台所有场景仅展示此昵称 |
| 随机头像 | 基于用户 ID 生成唯一的几何图案头像（类似 GitHub identicon） |
| 换马甲 | 每次发帖时可选择"换一个马甲"，生成新的随机昵称和头像，与账号解耦 |
| 后台实名 | 管理员后台可查看发帖者真实账号（仅用于追溯违规行为），普通用户不可见 |

### 2.3 个人中心

| 功能 | 说明 |
|------|------|
| 我的发布 | 查看自己发布的所有帖子，支持删除操作 |
| 我的评论 | 查看自己的所有评论，可跳转到原帖 |
| 我的收藏 | 查看收藏的帖子列表 |
| 账号设置 | 修改密码、修改默认昵称、切换深色模式 |

> 个人中心仅自己可见，其他用户无法访问。

## 八、推荐系统

### 8.1 推荐策略（阶段一：MVP）

基于标签偏好的加权排序，实现成本低，效果可验证。

**用户偏好计算：**

记录用户对各标签的交互行为，计算标签偏好权重：

```
标签偏好分 = 点赞次数 × 2 + 评论次数 × 3 + 收藏次数 × 3 + 浏览次数 × 1
```

将各标签偏好分归一化为 0-1 的权重值，存入 `user_tag_preference` 表。

**推荐排序公式：**

```
推荐分 = 标签匹配度 × 0.4 + 热度分(归一化) × 0.3 + 时间衰减分 × 0.3

其中：
- 标签匹配度：该帖子标签对应的用户偏好权重（0-1）
- 热度分：(点赞×2 + 评论×3 + 收藏×1) 归一化到 0-1
- 时间衰减分：1 / (1 + 发布距今小时数 / 24)，越新分越高
```

**偏好更新：** 用户每次点赞/评论/收藏时实时更新对应标签的偏好分。

### 8.2 冷启动处理

| 场景 | 策略 |
|------|------|
| 新用户（无行为数据） | 退化为热门排序 |
| 新帖子（无互动数据） | 给予新帖 2 小时的曝光保护期，期间获得额外时间衰减加分 |

### 8.3 防信息茧房

推荐结果中混入 20%-30% 的随机内容，确保用户能看到不同标签的帖子。

### 8.4 推荐策略（阶段二：进阶，后续迭代）

- 对帖子文本做关键词提取（TF-IDF 或 Embedding）
- 计算用户历史交互帖子与候选帖子的文本相似度
- 引入协同过滤：喜好相似的用户看过的内容互相推荐
- 适合用户量增长到千人级别以上时启用

## 九、三级审核机制

### 9.1 审核流程

```
用户发布内容（帖子/评论）
        │
        ▼
┌────────────────┐
│  L1：敏感词过滤  │──── 命中硬拦截词 ──▶ 直接拦截，提示用户修改
│   （同步执行）   │──── 命中软标记词 ──▶ 放行，标记送 AI 复查
└───────┬────────┘
        │ 通过
        ▼
┌────────────────┐
│  L2：AI 审核    │──── high 风险 ──▶ 内容隐藏，进入"待人工审核"队列
│  （异步执行）   │──── medium/low ──▶ 内容展示，标记为"AI 存疑"，进入人工复查队列
└───────┬────────┘
        │ safe
        ▼
    正常发布展示
        │
        │  ← 用户举报 / 定期抽检
        ▼
┌────────────────┐
│  L3：人工审核   │──▶ 通过 / 下架 / 下架+禁言
│  （管理后台）   │
└────────────────┘
```

### 9.2 L1：敏感词过滤

| 项目 | 说明 |
|------|------|
| 触发时机 | 发帖、评论提交时同步执行，阻塞式 |
| 算法 | DFA（确定有限自动机），时间复杂度 O(n)，单次 <5ms |
| 变体检测 | 支持检测空格插入（"傻 B"）、符号替换（"s*b"）、谐音等常见绕过手段 |
| 词库分级 | `硬拦截`：直接阻止发布，高亮违规词提示修改；`软标记`：放行但标记给 L2 复查 |
| 词库管理 | 管理后台可增删改敏感词，支持批量导入 |

### 9.3 L2：AI 审核

| 项目 | 说明 |
|------|------|
| 触发时机 | 内容通过 L1 后异步执行，不阻塞发布体验 |
| 审核维度 | 人身攻击、色情暗示、广告引流、隐私泄露、自残轻生倾向 |
| 输出格式 | `{ "risk_level": "high/medium/low/safe", "reason": "...", "category": "..." }` |
| 图片审核 | 上传图片传入多模态模型检测违规视觉内容 |
| 成本控制 | 短文本（<50 字）且无图片，仅走 L1 + 规则，跳过 AI 调用 |
| 降级方案 | AI 接口超时或异常时，内容正常发布，自动加入人工审核队列 |

### 9.4 L3：人工审核

| 项目 | 说明 |
|------|------|
| 审核队列 | 优先级排序：用户举报 > AI 高风险 > AI 存疑 > 定期抽检 |
| 审核操作 | `通过`（确认无问题）/ `下架`（隐藏内容）/ `下架+禁言`（同时处罚用户） |
| 审核详情 | 展示原文、AI 审核结果及理由、举报次数及原因、用户历史违规记录 |
| 操作留痕 | 所有审核操作记录：审核人、时间、决定、原因 |

### 9.5 内容状态流转

```
normal       — 正常展示
ai_suspect   — AI 存疑，正常展示，等待人工复查
pending      — 待人工审核（AI 高风险），不展示
rejected     — 已下架
```

| 队列排序 | 用户举报 > AI 高风险 > AI 存疑 > 定期抽检 |
| 审核详情 | 原文内容、AI 审核结果、举报信息、用户违规历史 |
| 批量操作 | 支持批量通过、批量下架 |

### 14.4 举报管理

| 功能 | 说明 |
|------|------|
| 举报列表 | 展示所有举报记录，包含被举报内容、举报原因、举报时间 |
| 处理操作 | `下架内容`、`下架+禁言用户`、`忽略举报` |
| 处理结果 | 处理后自动通知举报者和被举报者 |

### 14.5 用户管理

| 功能 | 说明 |
|------|------|
| 用户列表 | 展示注册用户，包含注册时间、发帖数、违规次数、当前状态 |

### 14.6 敏感词管理

| 功能 | 说明 |
|------|------|
| 词库列表 | 展示所有敏感词及其级别（硬拦截/软标记） |
| 增删改 | 单条新增、编辑、删除 |
| 批量导入 | 支持 TXT 文件批量导入敏感词 |
| 分级设置 | 每个词可设置为"硬拦截"或"软标记" |

### 14.7 置顶与公告管理

| 功能 | 说明 |
|------|------|

| 公告管理 | 发布、编辑、删除系统公告，设置展示时间范围 |

## 十五、防滥用机制

### 15.1 频率限制

| 限制项 | 规则 |
|--------|------|
| 发帖 | 同一账号 1 分钟内最多 1 帖，每天最多 20 帖 |
| 评论 | 同一账号 1 分钟内最多 5 条评论 |
| 私信 | 同一账号对同一楼主每天最多 3 条私信 |
| 举报 | 同一用户对同一内容仅可举报 1 次 |
| 搜索 | 同一 IP 每分钟最多 30 次搜索请求 |

### 15.2 账号限制

| 场景 | 处理 |
|------|------|
| 未登录用户 | 可浏览内容，不可发帖/评论/点赞/私信 |
| 禁言用户 | 可浏览内容，不可发帖/评论/私信，可点赞和收藏 |

### 15.3 隐私保护

| 措施 | 说明 |
|------|------|
| 身份隔离 | 用户 ID 与帖子通过中间映射关联，API 层不返回任何真实身份信息 |
| 图片隐私 | 上传图片自动去除 EXIF 信息 |
| IP 不暴露 | 前台不展示用户 IP 信息 |
